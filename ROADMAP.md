# 登录插件项目进度安排

## 项目信息

- 项目定位：为中小型项目提供可插拔、可配置、可扩展的登录插件，尽可能降低接入成本。
- 第一阶段目标：先完成 6 种常见登录方式的统一接入能力。
- 首期支持方式：
  - 用户名/邮箱/手机号 + 密码
  - 短信验证码登录
  - 邮箱验证码 / 魔法链接登录
  - 企业微信登录
  - 飞书登录
  - 微信登录
- 推荐 GitHub 仓库名：`omni-login-kit`

## 总体原则

- 优先做 MVP，不一开始追求“覆盖所有框架、所有平台、所有协议”。
- 优先解决“接入统一、配置统一、账号统一、文档清晰”。
- 插件设计优先于单一登录方式实现，先搭底座再扩 Provider。
- 先保证 Node.js + TypeScript 生态可用，再考虑多框架适配。

## 第一版暂不做

- 不做全框架兼容
- 不做 SAML / LDAP / 企业级身份编排
- 不做多租户
- 不做后台管理系统完整版
- 不做所有短信/邮件服务商全接入
- 不做所有微信生态形态全覆盖

## 我对项目节奏的建议

- 先用 2 到 3 周做出核心骨架和前 3 种登录方式
- 再用 1 到 2 周接入 企业微信和飞书
- 最后集中处理短信、微信、账号绑定和文档
- 每完成一种登录方式，就立刻补示例与测试，不要最后再补
